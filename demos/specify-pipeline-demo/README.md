# Specify Pipeline Demo

Tests the full **Prometheus → Specify pipeline** integration in Atlas.

When you ask `@Atlas` to plan a feature, `Prometheus` should orchestrate the complete spec-kit chain — SpecifyConstitution → SpecifySpec → SpecifyPlan → SpecifyTasks → SpecifyAnalyze — before handing off to `Sisyphus` for implementation. This demo verifies that the pipeline runs correctly and produces well-structured artifacts.

---

## Quick Start

### Step 1 — Run baseline (unit tests pass immediately)

```bash
cd demos/specify-pipeline-demo
py -m unittest -v
```

Expected: **19 unit tests pass**, integration tests **SKIPPED** with a helpful message showing what to do next.

### Step 2 — Run the Specify pipeline via Atlas

Open VS Code Copilot Chat and paste the prompt from [DEMO_PROMPT.md](DEMO_PROMPT.md).

Choose **Option B** (planning only, faster) or **Option A** (full planning + implementation).

### Step 3 — Verify artifacts

```bash
py -m unittest -v
```

- After **Option B**: `TestPipelineIntegration` (7 tests) should now pass.
- After **Option A**: All test classes including `TestImplementationSmoke` pass.

### Step 4 — Validate artifacts manually (optional)

```bash
python spec_validator.py .specify/specs/expense-tracker/
```

| `TestValidateFile` | Always | File-not-found handling |

---

## What the Pipeline Builds

The demo asks Atlas to build an **expense tracker CLI** using the Specify pipeline:

```
python expenses.py add 15.50 food "Lunch"
python expenses.py list --category food
python expenses.py summary
```

This is intentionally simple so the focus stays on validating the pipeline, not the feature.
